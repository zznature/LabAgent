# Canonical spectra require explicit scientific axes

A `raman-spectrum` canonical artifact is valid only when every scientific axis has an explicit kind and unit. Runtime must not infer Raman shift, wavelength, pixel index, intensity units, or other scientific meaning from column position alone. Axis semantics may come from explicit source headers or from the trusted configuration of the instrument Adapter that defines a headerless export format. If neither source evidence nor Adapter configuration provides sufficient semantics, the source artifact remains preserved and the canonical artifact becomes `failed` with an observable normalization error.
